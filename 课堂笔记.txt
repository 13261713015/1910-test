git之版本控制器

1. 定义: git是一个BS架构的版本控制器，用户只需要安装客户端即可，服务端在浏览器中

         通过访问github官网进行操作。

2. 作用: 当多个用户操作同一个文件时，为了避免出现后一个用户会覆盖前一个用户文件的

         内容，需要通过版本控制器来对文件的版本进行管理，防止出现覆盖新乡，造成
	 
	 文件内容丢失。

3. 操作步骤:

   3.1 首先通过访问github的官网来创建一个资源库。

   3.2 然后在本地安装一个git的客户端。

   3.3 安装完成后，在需要进行下载文件的目录中右键选择“git bash here”打开命令窗口。

   3.4 通过git clone （github资源库路径）命令将资源库下载到本地。

   3.5 下载完资源库后，将需要上传至资源库的文件放在下载下来的文件夹中。

   3.6 继续通过git命令窗口进行操作。

   3.7 上传文件前，通过git init 命令进行git初始化。

   3.8 初始化完成后，通过git add (上传的文件的名称)将即将上传的文件加载到git中。

   3.9 通过git commit -m “文件上传的备注信息”将文件上传的备注信息进行填写，首次

       上传时，需要运行两行命令输入邮箱地址和用户名来验证你的身份。

   3.10 通过git remote add origin "要上传的资源库路径"链接github上的资源库

   3.11 通过 git push -u origin master命令将git中的文件推送至github服务端。

   3.12 打开github中的资源库，查看文件上传是否成功。
 
   3.13 在文件通过git push命令在进行推送时，如果产生版本冲突问题，则需要通过
   
        git pull origin master命令将本地的资源库中的文件信息进行更新，更新完
	
	成后再次通过git push命令进行推送即可。
